Framework STEP — CICF

Como está organizada esta sessão

Esta atividade segue a Framework STEP do CICF — um ciclo de aprendizagem ativa com 5 momentos. Sabes sempre onde estás, o que se espera de ti e quanto tempo tens.

SETUP
"Preparar o Terreno"

Vibe Coding — Falar com a IA

Anatomia de um bom prompt

Pedir à IA para criar um jogo é como dar instruções a um programador muito rápido que nunca te viu trabalhar. Se fores vago, ele inventa. Se fores preciso, ele entrega. Vibe coding é exatamente isto — guiar a IA com palavras até ter um produto real.

🎵 O que é Vibe Coding?
É uma forma de programar onde descreves o que queres sentir e ver — o comportamento, o visual, a experiência — e a IA transforma as tuas palavras em código. Não precisas de saber sintaxe. Precisas de saber observar, descrever e avaliar.

Um bom prompt tem sempre 5 ingredientes. Podes não precisar de todos — mas quanto mais usares, melhor o resultado:

1
CONTEXTO
Onde estamos? Qual é o projeto?
→ "Estou a criar um jogo de browser em HTML num único ficheiro."
2
PAPEL DA IA
Diz à IA quem ela é neste momento
→ "Age como um developer experiente em jogos 2D para browser."
3
PEDIDO
CLARO
O que queres, com detalhe visual e de comportamento
→ "Cria um retângulo laranja de 200px que entre pelo lado esquerdo, se mova para a direita a 3px por frame, e pare quando carrego ESPAÇO."
4
FORMATO
DO OUTPUT
Como queres receber o resultado
→ "Devolve apenas o código HTML completo, sem explicações à volta."
5
RESTRIÇÕES
O que NÃO pode acontecer
→ "Não uses libraries externas. Tudo tem de funcionar offline num ficheiro só."
// CONTEXTO Estou a criar um jogo de browser chamado STACK! em HTML puro (único ficheiro .html com Canvas 2D). // PAPEL Age como developer sénior de jogos 2D para browser. // PEDIDO Quero um retângulo colorido (200px × 40px) que: — entre pela esquerda do ecrã e se mova para a direita — quando carrego ESPAÇO ou faço TAP, para nesse sítio — nasce um novo bloco do lado oposto, um bloco acima — se o novo bloco não sobrepuser o anterior: game over — a velocidade aumenta 0.2px a cada nível // OUTPUT Devolve o ficheiro HTML completo e funcional. Começa pelo loop de jogo básico — o visual vem depois.
🔁 O ciclo do vibe coding
1. Escreves o prompt → 2. A IA gera código → 3. Testas no browser → 4. Descreves o que está errado → volta ao 1

As 4 superpotências que precisas para ser bom em vibe coding — nenhuma delas é "saber programar":

👁
Observar
Consegues descrever exatamente o que está a acontecer no ecrã vs. o que querias que acontecesse?
✍️
Descrever
Consegues traduzir uma ideia visual em palavras precisas? "mais rápido" não chega — "3px por frame" sim.
🔬
Testar
Sabes tentar partir o que criaste? Um bom tester encontra o bug antes do utilizador.
🔁
Iterar
Falhar não é perder — é informação. Cada bug que encontras é um prompt melhor na próxima tentativa.
O Desafio

Do prompt ao produto em menos de 2 horas

Vais usar o Claude.ai como co-programador para construir um jogo de torre a funcionar diretamente no browser — sem instalar nada, sem saber programar. O segredo está em saber comunicar com a IA: escrever bons prompts, testar o resultado e iterar.

🎯 Objetivo
Criar uma versão jogável do STACK! — um jogo onde empilhas blocos para construir a torre mais alta possível. O Claude.ai escreve o código; o teu papel é guiá-lo com prompts precisos e garantir que o resultado funciona.
O Jogo — Especificação

Como funciona o STACK!

Antes de pedires à IA, precisas de perceber exatamente o que queres construir. Aqui está a especificação completa:

↔️
Movimento
Blocos entram pelo lado esquerdo ou direito, deslizando ao longo da base do ecrã
⏹️
Ação do Jogador
TAP no ecrã ou ESPAÇO para parar o bloco em movimento
📏
Régua de Pontuação
Barra na base com 50 pts no centro; pontos diminuem para as margens
🏗️
Torre
Cada bloco colocado fica ligeiramente mais pequeno; nasce acima do anterior
❌
Game Over
Se o novo bloco não sobrepuser o anterior, a torre colapsa — jogo termina
🚀
Dificuldade
A velocidade aumenta gradualmente com a altura da torre
💡 Dica de Especificação
Uma IA só constrói bem o que entende bem. Antes de escrever o teu primeiro prompt, pega numa folha e desenha o ecrã do jogo. Isso vai tornar os teus prompts muito mais precisos.
Escolha de Tecnologia — MVP

Qual stack usar para o teu jogo?

Uma das primeiras decisões no desenvolvimento é escolher a tecnologia. Para este desafio, o critério principal é: velocidade até ter algo a funcionar no browser. Aqui estão as opções, ordenadas por facilidade:

Tecnologia O Que É Prós para este desafio Contras Dificuldade
HTML + CanvasRECOMENDADO Um ficheiro .html com JavaScript nativo e Canvas 2D Zero instalações. Abre no browser. A IA gera num único bloco de código. Fácil de copiar e testar. Sem framework; mais difícil de escalar além do MVP ● Fácil
PWA (HTML + Manifest)RECOMENDADO HTML/Canvas com um ficheiro JSON de manifesto para instalar no telemóvel Funciona offline. Pode ser "instalado" no telemóvel. Ainda é um ficheiro só. Ligeiramente mais configuração ● Fácil
React (CDN) React carregado via CDN — sem build step Componentes reutilizáveis. A IA conhece bem React. Mais verboso para jogos simples com Canvas ●● Médio
Phaser.js Framework de jogos 2D para browser Física, colisões e animações já incluídas Curva de aprendizagem maior. A IA pode gerar erros mais difíceis de debugar ●● Médio
React + Vite React com ambiente de desenvolvimento local Developer experience profissional Precisa de Node.js instalado. Não é adequado para 60 minutos ●●● Avançado
Unity / Godot Engines de jogos completas Profissional, exporta para web Fora de âmbito para esta atividade. Horas/dias de setup ●●● Avançado
✅ Decisão para este desafio
Usa HTML único com Canvas API + PWA manifest embutido. O Claude.ai gera tudo num único bloco. Copias, colas num ficheiro .html, e abre no browser — ou partilhas via URL com GitHub Pages ou itch.io. Não precisas de instalar nada.
Guia de Prompting

Como comunicar com a IA

O Claude.ai funciona como um colega de programação extremamente capaz — mas só produz bom código se receber instruções claras. Aqui estão os princípios:

1

Contexto primeiro, pedido depois

Começa sempre por explicar o que é o projeto, depois o que queres. A IA usa o contexto para tomar decisões melhores.

// ❌ Mau prompt
Faz um jogo de blocos.

// ✅ Bom prompt
Quero criar um jogo de browser em HTML puro (um único ficheiro .html com Canvas 2D).
O jogo chama-se STACK! e funciona assim:
— Um retângulo colorido entra pelo lado esquerdo ou direito do ecrã, a mover-se horizontalmente na base
— O jogador carrega ESPAÇO ou faz TAP para o parar
— Nasce um novo bloco ligeiramente mais pequeno, do lado oposto, um bloco acima
— Se o novo bloco não sobrepuser o anterior, game over
— A velocidade aumenta a cada nível
— Existe uma régua de pontuação na base com 50pts no centro
Começa com a estrutura HTML e o loop de jogo básico.
2

Itera em partes, não tudo de uma vez

Pede uma funcionalidade de cada vez. É mais fácil corrigir uma parte pequena do que reescrever tudo.

// Depois do código base funcionar, pede melhorias:

O jogo está a funcionar mas quero melhorar o visual:
1. Adiciona um efeito de partículas quando o bloco é colocado
2. O bloco em movimento deve ter uma sombra de "rastro"
3. Quando a pontuação é perfeita (bloco centrado), faz um flash amarelo no ecrã
Mantém tudo no mesmo ficheiro HTML.
3

Quando algo não funciona, descreve o comportamento errado

Nunca digas apenas "não funciona". Explica o que vês vs. o que esperavas.

// ❌ Inútil
O jogo está partido.

// ✅ Útil
Tenho um problema: quando carrego ESPAÇO, o bloco para mas o próximo bloco
aparece na posição errada — começa fora do ecrã pela esquerda em vez de pela direita.
A lógica de direção do novo bloco parece estar invertida. Podes corrigir?

QA — Quality Assurance

Testa como um profissional

QA (Quality Assurance) é o processo de testar um produto para encontrar erros antes que os utilizadores os encontrem. É uma carreira em si — e um mindset essencial em qualquer equipa de tech.

✓ Cenários a testar
  • Jogar normalmente até game over
  • Testar no desktop e no telemóvel
  • Testar no Chrome, Firefox e Safari
  • Clicar muito rapidamente (stress test)
  • Deixar o jogo parado sem interação
  • Verificar se o recorde é guardado após fechar o browser
✗ Erros comuns a procurar
  • Bloco passa a barreira lateral sem inverter
  • Pontuação negativa ou NaN
  • Torre não sobe corretamente no ecrã
  • Game over prematuro (bloco sobrepõe mas termina)
  • Velocidade não aumenta com o nível
  • Botão "jogar de novo" não reinicia tudo

IA Ética

Usar IA com responsabilidade

A IA é uma ferramenta poderosa. Como qualquer ferramenta, pode ser usada bem ou mal. Estas são as práticas que distinguem um utilizador responsável:

✓ Boas práticas
  • Verificar sempre o código antes de publicar — a IA pode errar
  • Ser transparente quando o código foi gerado por IA
  • Usar IA para aprender, não para substituir o pensamento crítico
  • Não partilhar dados pessoais ou confidenciais com a IA
  • Questionar o output: "faz sentido? funciona? é seguro?"
  • Dar crédito a ferramentas e recursos usados
✗ O que evitar
  • Copiar código sem perceber o que faz
  • Usar IA para criar conteúdo enganoso ou prejudicial
  • Tratar o output da IA como verdade absoluta
  • Depender 100% da IA sem desenvolver competências próprias
  • Ignorar erros de segurança no código gerado
  • Plagiar trabalho de outros via IA
⚠️ Ponto de Reflexão
A IA não substitui a tua capacidade de pensar. O que ela faz é amplificar o que já sabes — ou o que ainda não sabes, mas sabias perguntar. O teu valor como profissional está em saber fazer as perguntas certas.

Missão Bónus — Põe o Jogo Online

Do ficheiro local para o mundo

O teu jogo funciona no browser local — mas para o instalar como PWA no telemóvel ou partilhar com amigos, precisas de o publicar online com HTTPS. Aqui estão os dois caminhos mais rápidos, ambos gratuitos.

▲ Netlify Drop
⏱ 2 minutos · sem conta obrigatória

O método mais rápido do mundo. Arrasta a pasta, fica online de imediato.

1
Abre o browser e vai a netlify.com/drop
2
Cria uma pasta no computador chamada meu-jogo. Coloca o ficheiro jogo.html dentro e renomeia-o para index.html — o Netlify serve este ficheiro automaticamente.
3
Arrasta a pasta meu-jogo para a zona a tracejado na página do Netlify Drop.
4
Aguarda ~10 segundos. O Netlify gera um URL automático tipo https://nome-aleatorio.netlify.app
5
Copia o URL e abre no telemóvel.
Chrome Android: menu ⋮ → "Adicionar ao ecrã inicial"
Safari iOS: botão partilhar ↑ → "Adicionar ao ecrã de início"
6
Opcional: cria conta gratuita para manter o URL permanente e poder atualizar o jogo.
✅ Resultado
URL público com HTTPS em menos de 2 minutos. Qualquer pessoa com o link pode jogar. Funciona como PWA instalável.
⬡ GitHub Pages
⏱ 5–10 minutos · conta GitHub necessária

O método "real" dos developers. Fica permanente, ligado ao teu portfólio e com histórico de versões.

1
Cria conta gratuita em github.com se ainda não tens.
2
Clica no botão verde "New" para criar um repositório. Dá-lhe o nome meu-jogo-stack. Deixa como Public. Clica "Create repository".
3
Na página do repositório clica em "uploading an existing file". Arrasta o teu jogo.html para a zona de upload e renomeia-o para index.html no campo que aparece.
4
Em baixo, em "Commit changes", escreve "primeiro commit" e clica "Commit changes".
5
Vai a Settings → Pages (menu lateral). Em Source escolhe "Deploy from a branch". Em Branch seleciona main e pasta /(root). Clica Save.
6
Aguarda 1–2 minutos. O teu jogo fica disponível em:
https://[username].github.io/meu-jogo-stack
7
Para atualizar: volta ao repositório → clica no ficheiro → ícone do lápis ✏️ → cola o novo código → Commit. O site atualiza em ~1 minuto.
💡 Bónus de Carreira
O teu repositório GitHub é o teu portfólio de developer. Cada projeto que colocas ali é prova concreta do que sabes fazer — mais valiosa do que qualquer certificado.
🔒 Porque preciso de HTTPS para o PWA?
O browser só permite instalar PWAs em páginas servidas via HTTPS — uma ligação segura e encriptada. Ficheiros locais (file://) não contam. Tanto o Netlify como o GitHub Pages ativam HTTPS automaticamente e de graça.
